## Fix 1: Path Encoding — `encode_cwd()` (BUG)

**Problem:** `encode_cwd()` replaces only `/` with `-`. Claude Code's `sanitizePath()` replaces ALL non-alphanumeric characters with `-`. Paths containing underscores, dots, or spaces produce wrong directory names, and the forked session won't be found by `claude --resume`.

**Current:**
```python
def encode_cwd(path: str) -> str:
    return path.replace("/", "-")
```

**Fix:**
```python
import re

MAX_SANITIZED_LENGTH = 200

def encode_cwd(path: str) -> str:
    """Encode a directory path the way Claude Code does.
    Matches sanitizePath() in sessionStoragePortable.ts:311-319.
    
    Every non-alphanumeric character becomes '-'. Paths > 200 chars
    are truncated + djb2 hash suffix. The encoding is one-way — there
    is no decode function in Claude Code or here.
    """
    sanitized = re.sub(r'[^a-zA-Z0-9]', '-', path)
    if len(sanitized) <= MAX_SANITIZED_LENGTH:
        return sanitized
    hash_val = _djb2_hash(path)
    return f"{sanitized[:MAX_SANITIZED_LENGTH]}-{hash_val}"

def _djb2_hash(s: str) -> str:
    """djb2 hash matching Claude Code's simpleHash() fallback.
    Source: sessionStoragePortable.ts simpleHash function.
    Note: Claude Code uses Bun.hash() in Bun builds, which produces
    DIFFERENT hashes. findProjectDir() handles this with prefix fallback.
    """
    h = 5381
    for c in s:
        h = ((h << 5) + h + ord(c)) & 0xFFFFFFFF
    return format(h, 'x')
```

**Behavioral notes:**
- `re.sub(r'[^a-zA-Z0-9]', '-', ...)` replaces each character individually, NOT collapsing runs. `/srv/my__project` → `-srv-my--project` (two consecutive dashes). This matches the TS behavior where `.replace(/[^a-zA-Z0-9]/g, '-')` does per-character replacement.
- Unicode: the TS regex `[^a-zA-Z0-9]` does not match Unicode letter/digit classes. Non-ASCII characters become `-`. Python's `re.sub` with the same pattern behaves identically (no `re.UNICODE` flag).
- Leading `-` from root `/` is correct and expected: `/root` → `-root`.

**Migration strategy for existing directories:**

Existing on-disk directories created with the old encoding (e.g., `-srv-my_project` instead of `-srv-my-project`) will NOT match the new encoding. This affects:
- `cmd_list` — existing sessions in old-encoded dirs won't appear
- `cmd_fork` — writing to new-encoded dir when old one exists

Resolution: add a `_find_project_dir_compat()` helper that tries new encoding first, falls back to legacy `path.replace("/", "-")`:

```python
def _find_project_dir_compat(path: str, projects_dir: Path) -> Path | None:
    """Find project dir with backward compatibility for old encoding."""
    new_encoded = projects_dir / encode_cwd(path)
    if new_encoded.is_dir():
        return new_encoded
    
    legacy_encoded = projects_dir / path.replace("/", "-")
    if legacy_encoded.is_dir():
        return legacy_encoded
    
    return None
```

This avoids a reindex/migration step — old dirs are found transparently. New forks use the correct encoding.

**Risk:** Low for code change. Medium for migration — mitigated by compat fallback.

**Golden tests required:**

## Fix 2: Drop `decode_cwd()` — Read CWD from JSONL

**Problem:** `decode_cwd()` tries to reverse a one-way encoding. It produces wrong results for any path with dashes in the name (e.g., `PHAT-TOAD-with-Trails`). Claude Code has no decode function.

**Fix:** Replace with `get_cwd_from_jsonl()`:

```python
def get_cwd_from_jsonl(jsonl_path: Path, max_lines: int = 50) -> str:
    """Read the original CWD from the JSONL records.
    
    Scans up to max_lines for the first record with a truthy 'cwd' field.
    Non-transcript entries (summary, tag, etc.) may not have cwd, so we
    scan past them.
    
    Returns empty string if no cwd found (display as directory name fallback).
    """
    with open(jsonl_path) as f:
        for i, line in enumerate(f):
            if i >= max_lines:
                break
            try:
                obj = json.loads(line)
                cwd = obj.get("cwd")
                if cwd:
                    return cwd
            except json.JSONDecodeError:
                continue
    return ""
```

**Performance for `cmd_list`:** This reads up to 50 lines per session file. For the `list` command scanning many projects, cache the result — the CWD doesn't change within a session:

```python
_cwd_cache: dict[str, str] = {}

def get_cwd_cached(jsonl_path: Path) -> str:
    key = str(jsonl_path)
    if key not in _cwd_cache:
        _cwd_cache[key] = get_cwd_from_jsonl(jsonl_path)
    return _cwd_cache[key]
```

**Fallback:** When CWD is empty (empty file, no cwd field in first 50 lines), display the directory name as-is: `"(unknown: {dirname})"`.

**Remove:** Delete `decode_cwd()` entirely. Update all call sites:
- `cmd_list`: `decode_cwd(proj_dir.name)` → `get_cwd_cached(jsonl)`
- `cmd_fork`: `decode_cwd` in error messages → `get_cwd_from_jsonl`

> **Reviewer notes (Fix 2)**  
> - **Cache key:** use `resolved = jsonl_path.resolve()` as key to avoid duplicate cache entries for `../` variants.  
> - **Cache invalidation:** if fork/list runs in same process after file rewrite, cache could be stale — document “process-lifetime cache only” or invalidate on path operations touching that JSONL.  
> - **`max_lines` 50:** reasonable; if cwd is only on line 200 (corrupt ordering), user sees fallback — acceptable edge case; optional WARN log in debug mode.  
> - **Fallback string:** `"(unknown: {dirname})"` — ensure dirname is **encoded dir name** (safe) not raw path if it ever includes user data.

## Fix 3: Session Discovery — Add Prefix Fallback

**Problem:** Long paths (>200 chars) get hash suffixes that differ between Bun and Node.js builds of Claude Code. Our tool uses Python (djb2). If the hashes don't match, session not found.

**Fix:** Extract shared helper used by both `find_session()` and any future project dir lookup:

```python
def find_project_dir(project_path: str, user: str = None) -> Path | None:
    """Find the project directory with prefix fallback for long paths.
    
    1. Try exact match with current encode_cwd()
    2. Try legacy encoding (slash-only replace) for backward compat
    3. For long paths (>200 chars), try prefix match (hash may differ)
    
    Returns None if no match. Raises ValueError if multiple prefix matches
    (ambiguous — caller should present choices).
    """
    projects = get_projects_dir(user)
    if not projects or not projects.exists():
        return None
    
    # Exact match — new encoding
    exact = projects / encode_cwd(project_path)
    if exact.is_dir():
        return exact
    
    # Legacy compat — old slash-only encoding
    legacy = projects / project_path.replace("/", "-")
    if legacy.is_dir():
        return legacy
    
    # Prefix fallback — for long paths with different hash suffixes
    sanitized = re.sub(r'[^a-zA-Z0-9]', '-', project_path)
    if len(sanitized) <= MAX_SANITIZED_LENGTH:
        return None  # Short path, no prefix fallback needed
    
    prefix = sanitized[:MAX_SANITIZED_LENGTH]
    matches = [d for d in projects.iterdir() if d.is_dir() and d.name.startswith(prefix)]
    
    if len(matches) == 1:
        return matches[0]
    if len(matches) > 1:
        raise ValueError(f"Ambiguous prefix match for '{project_path}': {[m.name for m in matches]}")
    
    return None
```

**Naming:** `find_project_dir()` is the single source of truth for project directory resolution. `find_session()` uses it internally for per-directory scanning. No duplicate logic.

**Ambiguity:** Multiple prefix matches raise `ValueError` — the caller (`find_session`, `cmd_fork`) catches it and presents the matches as choices.

## Fix 4: Re-stamp Session Fields During Fork

**Problem:** claude-forker only rewrites `cwd`. Claude Code re-stamps `sessionId`, `cwd`, and other session fields.

**Fix:**

```python
def restamp_record(obj: dict, target_cwd: str, fork_id: str) -> dict:
    """Re-stamp session fields on a copy of the record.
    
    Returns a new dict — does not mutate the input.
    Matches insertMessageChain behavior in sessionStorage.ts:1039-1063.
    """
    result = {**obj}
    
    # Always re-stamp sessionId — this is what makes the fork independent
    if "sessionId" in result:
        result["sessionId"] = fork_id
    
    # Re-stamp cwd if present and different
    if result.get("cwd") and result["cwd"] != target_cwd:
        result["cwd"] = target_cwd
    
    # Leave version, userType, entrypoint, gitBranch as-is — they document
    # the original session's environment, which is useful provenance.
    
    return result
```

Returns a new dict (no mutation of shared references). Only re-stamps `sessionId` and `cwd` — the functional fields. Other fields are provenance.

**CWD absent case:** If a record has no `cwd` field (some non-transcript entries), we don't add one — `if result.get("cwd")` is falsy for missing keys.

---

> **Reviewer notes (Fix 4)**  
> - **Shallow copy:** `{**obj}` is shallow — nested dicts/lists are still shared if mutated later; fork loop should treat each `result` as immutable after write or deep-copy if downstream mutates.  
> - **Fix 5 interaction:** `fork_record` in Fix 5 also sets `sessionId` and cwd — ensure **one** code path calls either `restamp_record` or inlined logic, not double-stamping with divergent rules.  
> - **Line numbers** in sessionStorage.ts — same “pin by commit” note as Fix 1.

## Fix 5: Entry Type Awareness

**Problem:** claude-forker treats all JSONL lines uniformly. It should distinguish chain participants (transcript messages that have parentUuid) from standalone entries (metadata, tags, snapshots).

**Fix:**

```python
# Types that participate in the parentUuid chain.
# Conservative allowlist — new unknown types are treated as non-chain
# (safe: they get sessionId re-stamped but no chain manipulation).
CHAIN_PARTICIPANT_TYPES = frozenset({"user", "assistant", "system", "attachment"})

def is_chain_participant(obj: dict) -> bool:
    return obj.get("type") in CHAIN_PARTICIPANT_TYPES

def fork_record(obj: dict, target_cwd: str, fork_id: str) -> dict:
    """Process a single JSONL record during fork.
    
    Chain participants get full re-stamping (sessionId + cwd).
    Non-chain entries get sessionId only (metadata must stay consistent
    but doesn't need cwd rewrite).
    """
    result = {**obj}
    
    if "sessionId" in result:
        result["sessionId"] = fork_id
    
    if is_chain_participant(result):
        if result.get("cwd") and result["cwd"] != target_cwd:
            result["cwd"] = target_cwd
    
    return result
```

**Forward compatibility:** Using an allowlist means new unknown entry types default to non-chain treatment. This is the safe direction — a misclassified chain participant that gets only `sessionId` re-stamped (but not CWD) will still load, just with the old CWD in some records (which is informational-only per our source verification). The alternative (denylist) risks breaking chain integrity for new types.

**CWD on non-chain entries:** Not rewritten. These are metadata (tags, titles, snapshots) — their `cwd` (if present) documents where they were created, which is valid provenance.

---

> **Reviewer notes (Fix 5)**  
> - **Appendix heuristic** (`uuid` + `parentUuid` as chain hint) is good — consider implementing as **fallback** when `type` is missing or unknown, to reduce misclassification pain.  
> - **Duplication vs Fix 4:** `fork_record` supersedes `restamp_record` for the fork path — pick one public API (`fork_record` only) or have `fork_record` call `restamp_record` for chain participants to avoid drift.  
> - **Metrics:** chain/non-chain counts for Fix 8 depend on exact same predicate — single function `is_chain_participant` in one module.

## Fix 6: Chain Integrity — Entry-Type-Aware Verification

**Problem:** `verify_chain()` silently skips records without `uuid`, which includes both legitimate non-chain entries AND potentially broken transcript messages.

**Fix:**

```python
def verify_chain(jsonl_path: Path) -> dict:
    entries = {}
    children = {}
    roots = 0
    non_chain = 0
    warnings = []
    bad_lines = 0
    
    with open(jsonl_path) as f:
        for line_num, line in enumerate(f, 1):
            try:
                obj = json.loads(line)
            except json.JSONDecodeError:
                bad_lines += 1
                continue
            
            if not is_chain_participant(obj):
                non_chain += 1
                continue
            
            uid = obj.get("uuid")
            if not uid:
                warnings.append(
                    f"Line {line_num}: chain participant (type={obj.get('type')}) missing uuid"
                )
                continue
            
            entries[uid] = True
            parent = obj.get("parentUuid")
            if parent is None:
                roots += 1
            else:
                children.setdefault(parent, []).append(uid)
    
    # Check for broken parent references
    broken = 0
    for parent_list in children.values():
        for child_uid in parent_list:
            pass  # parent is key, already in children dict
    # Actually check: does every parentUuid reference an existing entry?
    broken_parents = []
    with open(jsonl_path) as f:
        for line in f:
            try:
                obj = json.loads(line)
                if not is_chain_participant(obj):
                    continue
                parent = obj.get("parentUuid")
                if parent and parent not in entries:
                    broken += 1
                    broken_parents.append(parent[:12])
            except json.JSONDecodeError:
                pass
    
    has_children = set(children.keys())
    leaves = [uid for uid in entries if uid not in has_children]
    branches = sum(1 for kids in children.values() if len(kids) > 1)
    
    return {
        "entries": len(entries),
        "non_chain_entries": non_chain,
        "roots": roots,
        "leaves": len(leaves),
        "branches": branches,
        "broken": broken,
        "broken_parents": broken_parents[:5],
        "warnings": warnings,
        "bad_lines": bad_lines,
    }
```

Uses `is_chain_participant` from Fix 5 (single definition). Reports bad lines and warnings explicitly rather than silently skipping.

---

> **Reviewer notes (Fix 6)**  
> - **Pseudocode cleanup:** Remove the dead loop `for child_uid in parent_list: pass` — it signals “unfinished” to implementers.  
> - **Double file read:** second pass for broken parents is O(2n); acceptable for CLI; for huge JSONL, merge into one pass (track parent refs while building `entries`).  
> - **`broken` count:** current logic increments per **line** with bad parent — could overcount duplicates; define whether you want unique broken parent UUIDs vs line count.  
> - **`roots`:** multiple roots may be valid after compaction — document interpretation.  
> - **Tests:** golden JSONL fixtures: empty, non-chain only, broken parent, missing uuid on user message.
