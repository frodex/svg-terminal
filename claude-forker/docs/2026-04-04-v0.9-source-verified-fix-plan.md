## Fix 1: Path Encoding

### Problem

`encode_cwd()` replaces only `/` with `-`. Claude Code's `sanitizePath()` (function `sanitizePath` in `sessionStoragePortable.ts`, commit `a5d3e17`) replaces ALL non-alphanumeric characters with `-`. Paths with underscores, dots, or spaces produce wrong directory names. The forked session won't be found by `claude --resume`.

### Current code

```python
def encode_cwd(path: str) -> str:
    return path.replace("/", "-")
```

### Replacement

```python
import re

MAX_SANITIZED_LENGTH = 200

def encode_cwd(path: str) -> str:
    """Encode a directory path the way Claude Code does.
    
    Matches sanitizePath() in sessionStoragePortable.ts.
    Every non-alphanumeric character becomes '-' (per-character, no
    run collapsing). Paths > 200 chars are truncated + djb2 hash suffix.
    The encoding is one-way — there is no decode function.
    """
    sanitized = re.sub(r'[^a-zA-Z0-9]', '-', path)
    if len(sanitized) <= MAX_SANITIZED_LENGTH:
        return sanitized
    hash_val = _djb2_hash(path)
    return f"{sanitized[:MAX_SANITIZED_LENGTH]}-{hash_val}"


def _int_to_base36(n: int) -> str:
    """Convert non-negative integer to base-36 string (matching JS Number.toString(36))."""
    if n == 0:
        return "0"
    digits = "0123456789abcdefghijklmnopqrstuvwxyz"
    result = []
    while n > 0:
        result.append(digits[n % 36])
        n //= 36
    return "".join(reversed(result))


def _djb2_hash(s: str) -> str:
    """djb2 hash matching Claude Code's simpleHash() fallback.
    
    Source: hash.ts djb2Hash() + sessionStoragePortable.ts simpleHash().
    Initial value 0, formula (h << 5) - h + charCode, signed 32-bit
    wrapping via | 0 semantics, then abs(), then base-36 output.
    
    Claude Code uses Bun.hash() in Bun builds, which produces different
    hashes. find_project_dir() handles this mismatch with prefix fallback.
    """
    h = 0
    for c in s:
        h = ((h << 5) - h + ord(c))
        # Signed 32-bit wrapping (JavaScript's | 0)
        h = h & 0xFFFFFFFF
        if h >= 0x80000000:
            h -= 0x100000000
    return _int_to_base36(abs(h))
```

### Behavioral contract

- `re.sub(r'[^a-zA-Z0-9]', '-', ...)` replaces each character individually. No run collapsing. `/srv/my__project` → `-srv-my--project` (two consecutive dashes). Matches TS `.replace(/[^a-zA-Z0-9]/g, '-')`.
- Non-ASCII characters become `-`. The regex `[^a-zA-Z0-9]` is ASCII-only in both TS and Python (no `re.UNICODE`).
- Leading `-` from root `/` is correct: `/root` → `-root`.
- Paths > 200 chars: `sanitized[:200]` + `-` + `abs(djb2_hash(original_path))` in base-36.

### Migration

No reindex or manual migration. `find_project_dir()` (Fix 3) tries new encoding first, falls back to legacy `path.replace("/", "-")`. Old directories are found transparently. New forks use the correct encoding.

If both old-encoded and new-encoded directories exist for the same logical path (unlikely — requires manual copying), new-encoded wins.

## Fix 2: Drop decode_cwd — Read CWD from JSONL

### Problem

`decode_cwd()` reverses the encoding, but the encoding is lossy. `/srv/PHAT-TOAD-with-Trails` encodes to `-srv-PHAT-TOAD-with-Trails`, which `decode_cwd` turns into `/srv/PHAT/TOAD/with/Trails`. Claude Code has no decode function.

### Delete

```python
# DELETE ENTIRELY — no replacement function
def decode_cwd(encoded: str) -> str:
    ...
```

### Replacement

```python
def read_first_cwd_from_jsonl(jsonl_path: Path, max_lines: int = 50) -> str:
    """Read one representative CWD from the first JSONL record that has cwd set.
    
    Non-transcript entries (summary, tag, etc.) may not have a cwd field,
    so scan up to max_lines. Returns empty string if not found.
    
    Complements (does not replace) read_cwds_from_jsonl(), which aggregates
    all cwd values with counts for fork/session matching.
    """
    with open(jsonl_path) as f:
        for i, line in enumerate(f):
            if i >= max_lines:
                break
            try:
                obj = json.loads(line)
                cwd = obj.get("cwd")
                if cwd:
                    return cwd
            except json.JSONDecodeError:
                continue
    return ""


_read_first_cwd_cache: dict[str, str] = {}

def read_first_cwd_cached(jsonl_path: Path) -> str:
    """Process-lifetime cache over read_first_cwd_from_jsonl.
    
    CWD does not change within a session, so reading once is sufficient.
    Cache key uses resolved path to avoid duplicates from relative paths.
    Not invalidated — this is a read-only display cache.
    """
    key = str(jsonl_path.resolve())
    if key not in _read_first_cwd_cache:
        _read_first_cwd_cache[key] = read_first_cwd_from_jsonl(jsonl_path)
    return _read_first_cwd_cache[key]
```

### Call site updates

| Location | Current (as in `tools/claude-fork`) | Updated |
|----------|--------------------------------------|---------|
| `cmd_list` | `project_cwd = decode_cwd(proj_dir.name)` | `project_cwd = read_first_cwd_cached(jsonl)` |
| `find_session` (suggestion list) | `cwd = decode_cwd(proj_dir.name)` | `cwd = read_first_cwd_from_jsonl(jsonl)` (per JSONL; avoid decoding dir name) |

### Fallback

If `read_first_cwd_from_jsonl` returns empty string (empty file, no cwd in first 50 lines): display `"(dir: {proj_dir.name})"` — the raw encoded directory name. Safe for display, clearly signals the value is derived, not authoritative.

## Fix 3: Unified Directory Resolution

### Problem

Directory lookups are scattered across `find_session()` and `cmd_fork()` with different encoding logic. Need one function that handles: current encoding → legacy encoding → prefix fallback for long paths.

### New function

```python
def find_project_dir(project_path: str, user: str = None) -> Path | None:
    """Single source of truth for project directory resolution.
    
    Resolution order:
      1. Exact match with current encode_cwd() — the correct encoding
      2. Legacy match with path.replace("/", "-") — backward compat
      3. Prefix match for long paths (>200 chars) where hash suffix
         may differ between Bun/Node/Python builds
    
    Returns the matching directory Path, or None if not found.
    Raises ValueError if multiple prefix matches (ambiguous).
    """
    projects = get_projects_dir(user)
    if not projects or not projects.exists():
        return None
    
    # 1. Current encoding
    exact = projects / encode_cwd(project_path)
    if exact.is_dir():
        return exact
    
    # 2. Legacy encoding (pre-Fix-1 directories)
    legacy = projects / project_path.replace("/", "-")
    if legacy.is_dir():
        return legacy
    
    # 3. Prefix fallback (long paths only)
    sanitized = re.sub(r'[^a-zA-Z0-9]', '-', project_path)
    if len(sanitized) <= MAX_SANITIZED_LENGTH:
        return None
    
    prefix = sanitized[:MAX_SANITIZED_LENGTH] + "-"
    matches = [d for d in projects.iterdir()
               if d.is_dir() and d.name.startswith(prefix)]
    
    if len(matches) == 1:
        return matches[0]
    if len(matches) > 1:
        raise ValueError(
            f"Ambiguous prefix match for '{project_path}': "
            f"{[m.name for m in matches]}"
        )
    
    return None
```

### Where find_project_dir is used

`find_project_dir()` is used by `cmd_fork` to resolve the TARGET directory when the user says "fork session X to /srv/my-project". It answers: "where should the forked JSONL go?"

`find_session()` does NOT use `find_project_dir`. It scans all project directories because it doesn't know which project a given session ID belongs to. The full `find_session` implementation is in Fix 7 (which adds worktree fallback to the existing scan logic).

### ValueError handling

Callers catch `ValueError` and present the matches:

```python
try:
    target_dir = find_project_dir(target_cwd, target_user)
except ValueError as e:
    if args.json:
        print(json.dumps({"command": "fork", "status": "error", "exitCode": 3,
            "error": "Ambiguous project directory", "detail": str(e),
            "suggestions": []}, indent=2))
    else:
        print(f"Error: {e}")
    sys.exit(3)
```

## Fix 4+5: Record Processing (Merged)

### Problem

claude-forker treats all JSONL lines uniformly — only rewriting `cwd`. It should:
1. Distinguish chain participants from metadata entries
2. Re-stamp `sessionId` on all records (not just `cwd`)
3. Return new dicts (no mutation)

Fix 4 (re-stamp sessionId) and Fix 5 (entry type awareness) are merged into a single `fork_record()` function.

### New code

```python
CHAIN_PARTICIPANT_TYPES = frozenset({
    "user", "assistant", "system", "attachment"
})

def is_chain_participant(obj: dict) -> bool:
    """Transcript messages that participate in the parentUuid chain.
    
    Conservative allowlist — unknown types default to non-chain treatment.
    Secondary heuristic: if a record has both 'uuid' and 'parentUuid'
    fields, treat it as a chain participant even if its type is unknown.
    This provides forward compatibility with new Claude Code entry types.
    """
    if obj.get("type") in CHAIN_PARTICIPANT_TYPES:
        return True
    if obj.get("uuid") is not None and "parentUuid" in obj:
        return True
    return False


def fork_record(obj: dict, target_cwd: str, fork_id: str) -> dict:
    """Process a single JSONL record during fork. Returns a new dict.
    
    Chain participants: re-stamp sessionId + cwd.
    Non-chain entries: re-stamp sessionId only (cwd is provenance).
    Records without sessionId: pass through unchanged (copy only).
    """
    result = {**obj}
    
    if "sessionId" not in result:
        return result
    
    result["sessionId"] = fork_id
    
    if is_chain_participant(result):
        if result.get("cwd") and result["cwd"] != target_cwd:
            result["cwd"] = target_cwd
    
    return result
```

## Fix 6: Chain Verification

### Problem

`verify_chain()` silently skips records without `uuid`, doesn't distinguish chain participants from metadata, and reads the file twice.

### Replacement (single-pass)

```python
def verify_chain(jsonl_path: Path) -> dict:
    """Verify parentUuid chain integrity. Single-pass implementation.
    
    Returns a dict with:
      entries: count of chain participants with uuid
      nonChainEntries: count of non-chain records
      roots: entries with parentUuid == null (normal after compaction)
      leaves: entries not referenced as anyone's parent
      branches: parents with 2+ children (forks, streaming artifacts)
      broken: entries whose parentUuid references a non-existent uuid
      brokenParents: first 5 broken parent UUIDs (truncated for display)
      warnings: structural anomalies (chain participant missing uuid, etc.)
      badLines: lines that failed JSON parsing
    """
    entries = set()
    parent_refs = {}  # uuid -> parentUuid
    children = {}     # parentUuid -> [child uuids]
    roots = 0
    non_chain = 0
    warnings = []
    bad_lines = 0
    
    with open(jsonl_path) as f:
        for line_num, line in enumerate(f, 1):
            try:
                obj = json.loads(line)
            except json.JSONDecodeError:
                bad_lines += 1
                continue
            
            if not is_chain_participant(obj):
                non_chain += 1
                continue
            
            uid = obj.get("uuid")
            if not uid:
                warnings.append(
                    f"Line {line_num}: {obj.get('type', '?')} missing uuid"
                )
                continue
            
            entries.add(uid)
            parent = obj.get("parentUuid")
            parent_refs[uid] = parent
            
            if parent is None:
                roots += 1
            else:
                children.setdefault(parent, []).append(uid)
    
    # Broken: parentUuid points to a uuid not in entries
    broken_parents = []
    for uid, parent in parent_refs.items():
        if parent is not None and parent not in entries:
            broken_parents.append(parent[:12])
    broken = len(broken_parents)
    
    # Leaves: entries that no other entry claims as parent
    referenced_as_parent = set(children.keys())
    leaves = [uid for uid in entries if uid not in referenced_as_parent]
    
    # Branches: parents with 2+ children
    branches = sum(1 for kids in children.values() if len(kids) > 1)
    
    return {
        "entries": len(entries),
        "nonChainEntries": non_chain,
        "roots": roots,
        "leaves": len(leaves),
        "branches": branches,
        "broken": broken,
        "brokenParents": broken_parents[:5],
        "warnings": warnings,
        "badLines": bad_lines,
    }
```

### Interpretation guide

- **Multiple roots:** Normal after compaction. Each `compact_boundary` sets `parentUuid: null`.
- **Branches:** Most are streaming artifacts (1-2 entries deep). Deep branches (10+) indicate concurrent resume or deliberate forks.
- **Broken > 0:** Fork should fail (exit 5). The chain has orphaned records.
- **warnings:** Informational — logged but don't block the fork.
- **badLines:** Counted but don't block (the line is copied as-is).

## Fix 7: Worktree Fallback

### Problem

Claude Code's `resolveSessionFilePath()` checks sibling git worktrees when a session isn't found at the expected project directory. `find_session()` doesn't.

### Source-verified behavior (from `sessionStoragePortable.ts:403-466`)

There is NO special worktree directory. Worktree sessions use the same `~/.claude/projects/` structure as everything else. A worktree at `/srv/my-project/.claude/worktrees/feature-foo/` produces sessions in `~/.claude/projects/-srv-my-project--claude-worktrees-feature-foo/`.

The fallback in `resolveSessionFilePath` is:
1. Look in `~/.claude/projects/<sanitized-cwd>/` for the session
2. If not found, run `git worktree list --porcelain` to discover sibling worktrees
3. For each sibling, check `~/.claude/projects/<sanitized-sibling-path>/`

Worktree discovery source (`getWorktreePathsPortable.ts:12-27`):
```python
# Shells out to git, parses "worktree <path>" lines from porcelain output
git worktree list --porcelain
```

### New code

```python
import subprocess

def list_git_worktree_paths(repo_cwd: str) -> list[str]:
    """List absolute paths of all git worktrees (including main checkout).
    
    Wraps `git worktree list --porcelain` — same discovery as Claude Code's
    getWorktreePathsPortable. Returns empty list if git is unavailable,
    repo_cwd is not a repo, or the command times out.
    """
    try:
        result = subprocess.run(
            ["git", "worktree", "list", "--porcelain"],
            cwd=repo_cwd, capture_output=True, text=True, timeout=5
        )
        if result.returncode != 0:
            return []
        return [
            line[len("worktree "):]
            for line in result.stdout.split("\n")
            if line.startswith("worktree ")
        ]
    except (subprocess.SubprocessError, FileNotFoundError, OSError):
        return []
```

### Integration with find_session

Add worktree fallback after primary scan in `find_session()`:

```python
def find_session(session_id: str, user: str = None) -> tuple:
    projects = get_projects_dir(user)
    if not projects or not projects.exists():
        return None, None, []
    
    # Primary scan — check all project directories
    matches = []
    scanned_dirs = set()
    for proj_dir in sorted(projects.iterdir()):
        if not proj_dir.is_dir():
            continue
        scanned_dirs.add(proj_dir.name)
        for jsonl in proj_dir.glob("*.jsonl"):
            stem = jsonl.stem
            if stem == session_id:
                cwds = read_cwds_from_jsonl(jsonl)
                return jsonl, cwds, []
            if stem.startswith(session_id):
                matches.append((jsonl, proj_dir, stem))
    
    if len(matches) == 1:
        cwds = read_cwds_from_jsonl(matches[0][0])
        return matches[0][0], cwds, []
    elif len(matches) > 1:
        return None, None, matches
    
    # Worktree fallback — check sibling worktrees for sessions
    # that were started from a different worktree of the same repo
    cwd = os.getcwd()
    for wt_path in list_git_worktree_paths(cwd):
        encoded = encode_cwd(wt_path)
        if encoded in scanned_dirs:
            continue  # already checked in primary scan
        wt_proj_dir = projects / encoded
        if not wt_proj_dir.is_dir():
            # Try legacy encoding
            legacy = wt_path.replace("/", "-")
            if legacy in scanned_dirs:
                continue
            wt_proj_dir = projects / legacy
            if not wt_proj_dir.is_dir():
                continue
        for jsonl in wt_proj_dir.glob("*.jsonl"):
            stem = jsonl.stem
            if stem == session_id or stem.startswith(session_id):
                matches.append((jsonl, wt_proj_dir, stem))
    
    if len(matches) == 1:
        cwds = read_cwds_from_jsonl(matches[0][0])
        return matches[0][0], cwds, []
    elif len(matches) > 1:
        return None, None, matches
    
    # No matches — collect suggestions
    suggestions = []
    for proj_dir in sorted(projects.iterdir()):
        if not proj_dir.is_dir():
            continue
        for jsonl in proj_dir.glob("*.jsonl"):
            cwd_val = read_first_cwd_from_jsonl(jsonl)
            display = cwd_val if cwd_val else f"(dir: {proj_dir.name})"
            size = jsonl.stat().st_size
            mtime = datetime.fromtimestamp(jsonl.stat().st_mtime).strftime("%b %d")
            suggestions.append(f"{jsonl.stem} ({display}, {size // 1024}KB, {mtime})")
    return None, None, suggestions[:10]
```

### Behavior

- Worktree fallback only runs if primary scan finds nothing — no performance cost in the common case.
- Uses `os.getcwd()` as the git context for `git worktree list` — if the user isn't in a git repo, the command returns empty and the fallback is skipped silently.
- Checks both current and legacy encoding for each worktree path.
- Tracks `scanned_dirs` to avoid re-scanning directories already checked in the primary pass.

### NFC normalization divergence

Claude Code's `getWorktreePathsPortable` applies `.normalize('NFC')` to each worktree path. This Python implementation does not. On Linux with typical ASCII paths, this has no effect. If non-ASCII paths are encountered, normalize each path when building the list returned by `list_git_worktree_paths`. Document in `docs/migration.md`.

## Intentional Differences from Claude Code

### Raw copy vs chain-walk fork
Claude Code's `--fork-session` loads via parentUuid chain walk (drops pre-compact history), then re-serializes. Our fork preserves the full raw file including pre-compact history. **Intentional** — the agent-repo requires full archives. Materialization can prune to post-compact-only.

### CWD rewriting on chain records
Claude Code's loader does not check `cwd` values — discovery is directory-based only. Our CWD rewrite is technically unnecessary for loading but is kept because the `cwd` field is visible to the model as context and may be used by tools for path resolution.
