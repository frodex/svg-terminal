## PRD Summary — Key Takeaways

### Architecture
- **Rendering pipeline:** tmux -> server.mjs (30ms poll, line diff) -> WebSocket -> terminal.svg (SVG) -> `<object>` in card DOM -> CSS3DRenderer -> screen
- **4x scale trick:** Card DOM oversized, CSS3DObject scale=0.25. Forces Chrome high-res rasterization. DO NOT REMOVE.
- **Camera-only focus:** Cards never change size on focus. Camera moves instead. Eliminated an entire class of two-state sizing bugs. This was a hard-won architectural decision (abandoned DOM resize approach 2026-03-29).
- **Frustum-projected layout:** Multi-focus computes in screen pixels, projects into 3D. Cell-count proportional area allocation.

### Card System
- `createCardDOM(config)` — generic factory for terminal + browser cards
- Card sizing from `calcCardSize(cols, rows)` — uniform visual weight, different shapes
- During focus: card size unchanged, +/- changes cols/rows (font size), Alt+drag changes card DOM
- Two optimize directions: fit terminal to card, fit card to terminal

### Critical Constraints
- 4x scale trick (text blurs without it)
- Camera-only focus (no DOM resize)
- Coordinate-based hit testing (CSS3D ignores Z depth)
- No CSS border/box-shadow on .terminal-3d (triggers GPU re-rasterization)
- `tmux resize-window` not `resize-pane`

### Anti-Patterns (9 documented)
- CSS transform for font zoom -> change tmux cols/rows
- DOM resize on focus -> camera-only
- Camera offset for sidebar -> sidebar is overlay
- Card reshaping on +/- -> card stays, font changes
- `focusedSessions.clear()` on deselect -> keep intact
- `e.target.closest` in CSS3D -> coordinate-based rect checking
- Border/box-shadow active indicator -> header background only
- `_layoutZ` for Z-slide restore -> `_savedZ` captures current Z
- Floating overlay controls -> controls inline in header

## Key Protocol Differences (svg-terminal ↔ claude-proxy)

| Aspect | svg-terminal (current) | claude-proxy api-server |
|--------|----------------------|------------------------|
| WS endpoint | `/ws/terminal?session=X&pane=Y` | `/api/session/:id/stream` |
| Input format | `{ type: "input", keys: "hello" }` | `{ type: "input", keys: "hello" }` or `{ specialKey: "Enter" }` |
| Key translation | Browser key → tmux send-keys name | Browser key → ANSI escape code |
| Screen source | tmux capture-pane (30ms poll) | xterm/headless onRender events |
| SGR parsing | sgr-parser.mjs (client-side) | screen-renderer.ts (server-side, pre-parsed spans) |
| Session discovery | `/api/sessions` (tmux list-sessions) | `/api/sessions` (SessionManager, access-filtered) |
| Auth | None | OAuth + session cookies + middleware |
| Scroll | Not in WS (HTTP only) | `{ type: "scroll", offset: N }` in WS |
