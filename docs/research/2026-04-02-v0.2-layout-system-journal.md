### Phase 2: Card Sizing Investigation (deep dive)

Discovered the root cause of the letterbox gap, text blur, and cursor drift:

**Root cause:** Card DOM sized with hardcoded `SVG_CELL_W = 8.65` / `SVG_CELL_H = 17`, but SVG viewBox sized with runtime-measured `~8.61` / `~17.0`. The mismatch caused the SVG to letterbox inside the card (`preserveAspectRatio` default = `xMidYMid meet`).

**Key finding from runtime measurement:** Font metrics vary slightly per terminal (8.604â€“8.654 width, 16.93â€“17.85 height) due to sub-pixel rounding in `getBBox()` at different viewBox scales. The hardcoded constants were a "best-fit compromise" that didn't match any specific terminal exactly.

**First-principles conclusion:** One source of truth â€” measure font once at runtime, all components read from that measurement. The hardcoded constants are a workaround for a timing problem (card created before SVG measures font), not a design decision.

**Fix implemented:**
- `calcCardSize(cols, rows, cellW, cellH)` â€” accepts optional measured values
- `getMeasuredCellSize(t)` â€” reads actual font metrics from SVG `<object>`
- `updateCardForNewSize` uses measured values when available
- SVG load handler corrects card size after font measurement (100ms delay)
- `_needsMeasuredCorrection` flag ensures correction runs even if cols/rows unchanged
- All sizing paths prefer measured values, constants are fallbacks only

Debug process: checkerboard alignment tests (6 terminal sizes Ã— 6 mutation states), debug border overlays (red=card, white=object, green=viewBox), runtime measurement comparison.

## Dead Ends

1. **"Font-size mutation" as a separate lever** â€” Explored extensively, then realized there is no font-size lever. The SVG font-size is fixed (14px, monospace). Apparent font size is fully determined by card size, terminal size, and Z-depth. Removed from design doc in v4.

2. **`preserveAspectRatio="none"` to fix letterbox** â€” Tried, didn't fix gap or cursor drift. The gap is from the aspect mismatch between card and viewBox, not from how the SVG fits. Reverted immediately.

3. **Per-card `SVG_CELL_W/H`** â€” Proposed making cell dimensions per-card for font-size mutation. Wrong â€” the font is fixed, cell size is fixed, the constants are only used for card aspect calculation. The real fix is using measured values, not per-card constants.

4. **Double-Escape for deselect** â€” Implemented, then reverted. User wanted deselect to automatically return to group grid, not a double-tap mechanism.
