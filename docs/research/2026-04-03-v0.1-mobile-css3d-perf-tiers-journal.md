## Context

Performance tiers (full / reduced / minimal) were added to degrade visuals when GPU or frame time is stressed. Puppeteer tests use `--disable-gpu` (software WebGL), so they validate the software path and UI cycling, not real phone hardware. Phones keep hardware WebGL; proactive heuristics (`maybeProactiveAutoTier1`) bump **reduced** early on coarse pointer + landscape + many cards.

## Problem observed on phone

- Portrait could render; **landscape** with more visible cards caused the lower half of the screen to fail (GPU / compositor stress).
- After auto switched **full → reduced**, the experience was still broken: **focused cards seemed missing**, **sidebar thumbnails looked invisible**, **background cards flickered**.

## Root cause analysis

### Tier 2 (minimal) visibility bug

`syncPerfTier2Visibility` used:

`show = focusedSessions.size > 0 && focusedSessions.has(name)`

When **no session was focused** (overview), `focusedSessions.size === 0`, so **every** card had `visible = false`. The ring disappeared entirely — empty 3D view — which felt like “missing” focused content and broken thumbnails beside a blank scene. Integration tests had even encoded “all cards hidden in Tier 2 with no focus” as the expected behavior, which reinforced the wrong UX.

**Intended semantics:** minimal mode should **cull unfocused cards only while focus mode is active**. In **overview**, the full ring must remain visible so the user can navigate.

**Fix:** centralize with `tier2CardShouldShow(name)`:

`focusedSessions.size === 0 || focusedSessions.has(name)`

Apply consistently in `applyPerfTier`, `syncPerfTier2Visibility`, `addTerminal`, `addBrowserCard`, and replace the `unfocusTerminal` block that forced all objects invisible with `syncPerfTier2Visibility()`.

### “Reduced” still too heavy

Tier 1 turned off ring spin, shadows (group), and specular **display**, but the animation loop still every frame:

- Updated **shadow DOM** (blur, opacity, position) for every card
- Ran **float** animation on unfocused cards
- Ran **billboard slerp + lazy drift** on unfocused cards

On mobile, that still drives constant CSS3D / layout work and shows up as **flicker** and jank.

**Fix:** treat `perfTier >= 1` as **reduceMotion** for the 3D loop: no float, no drift, copy quaternion to billboard target (no slerp), and **skip shadow + specular updates entirely** (tier 0 only).

### Escalation when reduced is insufficient

Added `maybeProactiveAutoTier2()` (auto, from tier 1) for touch + high load + many cards / landscape. FPS auto-detect uses a **lower** Tier 2 threshold on coarse pointer (~42 ms vs 50 ms) so phones move to minimal sooner.

### Thumbnails “invisible”

Likely mix of empty scene (Tier 2 bug) and compositor stacking. **CSS:** `transform: translateZ(0)` + `isolation: isolate` on `.thumbnail-sidebar` to promote a stable layer.

## Tests

`test-performance-mode.mjs` Test 3 updated: Tier 2 in overview expects **zero** hidden cards (all ring cards visible). Documented limitations at top of test file (no real mobile GPU).

## Open questions / follow-ups

- Thermal / driver-specific limits may still cause partial repaint on some devices — may need device-specific profiling beyond heuristics.
- Whether to further lower Tier 2 FPS threshold or add explicit “low power” user toggle is product-dependent.
