# Admin Bypass + cp-users Gate for Session Visibility v0.2

**Preceding:** v0.1 (admin bypass only)
**Changes from v0.1:** Added `cp-users` as prerequisite gate for all non-admin, non-owner access. Users not in `cp-users` cannot see ANY sessions (including via allowedUsers/allowedGroups). Updated matrix and both code changes.

**Goal:** Fix two authorization issues in claude-proxy session visibility:
1. Root and admins cannot see sessions owned by other users (missing admin bypass)
2. Users not in `cp-users` group can still access sessions via `allowedUsers`/`allowedGroups` (missing baseline gate)

**Root cause:** `listSessionsForUser` (session-manager.ts:1067) and `canUserAccessSession` (user-utils.ts:78) have no admin bypass and no `cp-users` membership prerequisite.

## Visibility Matrix (after patch)

| User type | public, not hidden | public, hidden | private, in allowedUsers | private, in allowedGroups | private, no access |
|-----------|-------------------|----------------|--------------------------|---------------------------|--------------------|
| **root** | YES | YES | YES | YES | YES |
| **cp-admins member** | YES | YES | YES | YES | YES |
| **Owner** | YES | YES | YES | YES | YES |
| **cp-users member** | YES | NO | YES | YES | NO |
| **Not in cp-users** | NO | NO | NO | NO | NO |

**Rule:** `cp-users` membership is the prerequisite for all non-admin, non-owner session access. Without it, you are authenticated but not authorized to participate in any session.

## Change 1: `listSessionsForUser` (session-manager.ts)

**Before (line 1067-1083):**
```typescript
listSessionsForUser(username: string): Session[] {
    return Array.from(this.sessions.values()).filter(session => {
      // Socket-based sessions with provisioner: use group-based access control
      if (session.socketPath && this.provisioner) {
        if (username === session.access.owner) return true;
        if (session.access.hidden && username !== session.access.owner) return false;
        const groups = this.provisioner.getGroups(username);
        const sessionGroup = `sess-${session.name.replace(/[^a-zA-Z0-9_-]/g, '_')}`;
        if (session.access.public) {
          return groups.includes('users');
        }
        return groups.includes(sessionGroup) ||
               session.access.allowedGroups.some(g => groups.includes(g));
      }
      // Legacy sessions: use existing ACL check
      return canUserAccessSession(username, session.access);
    });
  }
```

**After:**
```typescript
listSessionsForUser(username: string): Session[] {
    const isAdmin = username === 'root' || isUserInGroup(username, 'admins');
    return Array.from(this.sessions.values()).filter(session => {
      // Admins and root see all sessions (including hidden)
      if (isAdmin) return true;
      // Owner always sees their own sessions
      if (username === session.access.owner) return true;
      // Socket-based sessions with provisioner: use group-based access control
      if (session.socketPath && this.provisioner) {
        if (session.access.hidden) return false;
        const groups = this.provisioner.getGroups(username);
        // cp-users membership is required for all non-admin, non-owner access
        if (!groups.includes('users')) return false;
        if (session.access.public) return true;
        const sessionGroup = `sess-${session.name.replace(/[^a-zA-Z0-9_-]/g, '_')}`;
        return groups.includes(sessionGroup) ||
               session.access.allowedGroups.some(g => groups.includes(g));
      }
      // Legacy sessions: use existing ACL check
      return canUserAccessSession(username, session.access);
    });
  }
```

**Changes:**
- Added `isAdmin` check before filter (root + cp-admins bypass)
- Moved owner check before socketPath branch (owner always sees own sessions regardless of path)
- Added `if (!groups.includes('users')) return false` gate before public/allowedGroups checks
- Reordered: hidden check before groups check (owner already handled above, so hidden = deny)

**Note:** `isUserInGroup` is already imported in session-manager.ts (line 13).

## Change 2: `canUserAccessSession` (user-utils.ts)

Used by: `getSessionScreen`, `forkSession`, `listDeadSessions`, legacy path of `listSessionsForUser`.

**Before (line 78-95):**
```typescript
export function canUserAccessSession(username: string, access: { owner: string; hidden: boolean; public: boolean; allowedUsers: string[]; allowedGroups: string[] }): boolean {
  // Owner always has access
  if (username === access.owner) return true;

  // Hidden sessions: only owner
  if (access.hidden) return false;

  // Public sessions: everyone
  if (access.public) return true;

  // Check allowed users
  if (access.allowedUsers.includes(username)) return true;

  // Check allowed groups
  for (const group of access.allowedGroups) {
    if (isUserInGroup(username, group)) return true;
```

**After:**
```typescript
export function canUserAccessSession(username: string, access: { owner: string; hidden: boolean; public: boolean; allowedUsers: string[]; allowedGroups: string[] }): boolean {
  // Admins and root access all sessions
  if (username === 'root' || isUserInGroup(username, 'admins')) return true;

  // Owner always has access
  if (username === access.owner) return true;

  // cp-users membership required for all non-admin, non-owner access
  if (!isUserInGroup(username, 'users')) return false;

  // Hidden sessions: only owner (owner already returned above)
  if (access.hidden) return false;

  // Public sessions: all cp-users members
  if (access.public) return true;

  // Check allowed users
  if (access.allowedUsers.includes(username)) return true;

  // Check allowed groups
  for (const group of access.allowedGroups) {
    if (isUserInGroup(username, group)) return true;
```

**Changes:**
- Added admin/root bypass at top
- Added `cp-users` gate after owner check, before any other access logic
- Updated comments for clarity

**Note:** `isUserInGroup` is defined in the same file. The `'users'` argument maps to `cp-users` system group (function prepends `cp-`).
