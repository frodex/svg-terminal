# Session Authorization & WebSocket Consolidation Implementation Plan v0.2

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Secure all session data paths with scoped API keys, consolidate HTTP endpoints onto the authorized WebSocket backchannel, harden dev mode, and add reconnection UX.

**Architecture:** Issue a scoped API key at login that identifies the user on all WebSocket connections. The key is stored server-side for instant revocation. All dashboard-to-server communication moves to the authenticated `/ws/dashboard` WebSocket. Claude-proxy remains the session permission authority; svg-terminal faithfully passes user identity on every RPC call. Staged rollout via `STRICT_SESSION_AUTHZ` feature flag.

**Tech Stack:** Node.js (server.mjs), vanilla JS (dashboard.mjs), SQLite (user-store.mjs), HMAC-SHA256 (session-cookie.mjs)

**Research:** `docs/research/2026-04-05-v0.1-session-authorization-journal.md`
**Reviewer notes:** `docs/research/2026-04-05-v0.1-session-authorization-journal-NOTES.md`
**Previous plan version:** `docs/superpowers/plans/2026-04-05-v0.1-session-auth-and-ws-consolidation.md`

## Phased Rollout

### Phase 1 — Session Authorization + Identity Fixes (Tasks 1-8)
- Feature flag `STRICT_SESSION_AUTHZ` (env var, default: off)
- When off: existing behavior (no per-session checks, backwards compatible)
- When on: all session data paths enforce authorization
- Rollback: set `STRICT_SESSION_AUTHZ=0`, restart

### Phase 2 — API Key Store + Credential Lifecycle (Tasks 9-15)
- Server-side key store with idle/absolute timeout
- Force re-login, admin PIN, rate limiting
- Reconnection overlay
- WS consolidation (move HTTP to WS)

## Threat Model

| Attacker | Capabilities | Mitigations |
|----------|-------------|-------------|
| Peer user (authenticated) | Valid OAuth, own API key, knows session names | Per-session authZ on every read/write path |
| Stolen cookie | Can call `/auth/api-key`, get keys | API key store with max 10 keys/user, revocable |
| Stolen API key | Can open WS, subscribe to sessions | One WS per key, server-side revocation, idle timeout |
| Leaked WS URL with token | Token in `<object>` URL visible in logs/Referer | Remove `/ws/terminal`, no more tokens in SVG URLs |
| Network observer | Sees tunnel traffic | TLS via Cloudflare tunnel, HttpOnly cookies |
| Compromised admin | Can approve/deactivate users | Admin PIN for privileged actions, sudo window, audit log |

- [ ] **Step 1: Add `authorizeSession()` to server.mjs**

This function checks if a user is allowed to access a specific session. When `STRICT_SESSION_AUTHZ` is off, it always returns true (backwards compatible).

```javascript
/**
 * Check if user is authorized to access a session.
 * @param {object} user - authenticated user (from getAuthUser or API key identity)
 * @param {string} sessionName - tmux/cp session name
 * @returns {boolean}
 */
function authorizeSession(user, sessionName) {
  if (!STRICT_SESSION_AUTHZ) return true;
  if (!user) return false;
  if (isSuperadmin(user)) return true;

  // For claude-proxy sessions: authorization is delegated to claude-proxy via RPC
  // (it will reject unauthorized requests). We just need to ensure we pass the right user.
  // For local tmux sessions: check session ownership cache.
  const cached = sessionPermCache.get(sessionName);
  if (!cached) return false; // unknown session = deny
  if (cached.public) return true;
  if (cached.owner === user.linux_user) return true;
  if (cached.allowedUsers && cached.allowedUsers.includes(user.linux_user)) return true;
  // Group check could be added here
  return false;
}

// Session permission cache — populated during session discovery
const sessionPermCache = new Map(); // sessionName → { owner, public, allowedUsers, allowedGroups, viewOnly }
```

```javascript
// api-key-store.mjs
import { randomBytes, createHmac } from 'node:crypto';

export class ApiKeyStore {
  constructor({ secret, idleTimeoutMs = 30 * 60 * 1000, absoluteTimeoutMs = 24 * 60 * 60 * 1000, maxKeysPerUser = 10 }) {
    this._secret = secret;
    this._idleTimeoutMs = idleTimeoutMs;
    this._absoluteTimeoutMs = absoluteTimeoutMs;
    this._maxKeysPerUser = maxKeysPerUser;
    this._keys = new Map();
    this._cleanupTimer = setInterval(() => this.cleanup(), 60000);
    if (this._cleanupTimer.unref) this._cleanupTimer.unref();
  }

  issue(email, linuxUser, browserUid) {
    // Enforce max keys per user — revoke oldest if over limit
    const userKeys = [];
    for (const [k, e] of this._keys) {
      if (e.email === email) userKeys.push({ key: k, issuedAt: e.issuedAt });
    }
    if (userKeys.length >= this._maxKeysPerUser) {
      userKeys.sort((a, b) => a.issuedAt - b.issuedAt);
      for (let i = 0; i <= userKeys.length - this._maxKeysPerUser; i++) {
        this._keys.delete(userKeys[i].key);
      }
    }

    const raw = randomBytes(32).toString('base64url');
    const hmac = createHmac('sha256', this._secret).update(raw).digest('base64url');
    const key = raw + '.' + hmac;
    this._keys.set(key, {
      email, linuxUser,
      browserUid: browserUid || null,
      issuedAt: Date.now(),
      lastActivity: Date.now(),
      wsClaimed: false,
    });
    return key;
  }

  validate(key) {
    const entry = this._keys.get(key);
    if (!entry) return null;
    const [raw, hmac] = key.split('.');
    if (!raw || !hmac) return null;
    const expected = createHmac('sha256', this._secret).update(raw).digest('base64url');
    if (hmac !== expected) { this._keys.delete(key); return null; }
    if (Date.now() - entry.lastActivity > this._idleTimeoutMs) { this._keys.delete(key); return null; }
    if (Date.now() - entry.issuedAt > this._absoluteTimeoutMs) { this._keys.delete(key); return null; }
    return { email: entry.email, linuxUser: entry.linuxUser, browserUid: entry.browserUid };
  }

  touch(key) {
    const entry = this._keys.get(key);
    if (entry) entry.lastActivity = Date.now();
  }

  claimWs(key) {
    const entry = this._keys.get(key);
    if (!entry) return false;
    if (entry.wsClaimed) return false; // duplicate connection
    entry.wsClaimed = true;
    return true;
  }

  releaseWs(key) {
    const entry = this._keys.get(key);
    if (entry) entry.wsClaimed = false;
  }

  revoke(key) { this._keys.delete(key); }

  revokeAllForUser(email) {
    for (const [key, entry] of this._keys) {
      if (entry.email === email) this._keys.delete(key);
    }
  }

  listForUser(email) {
    const result = [];
    for (const [key, entry] of this._keys) {
      if (entry.email === email) {
        result.push({ key: key.slice(0, 8) + '...', browserUid: entry.browserUid,
          issuedAt: entry.issuedAt, lastActivity: entry.lastActivity, wsClaimed: entry.wsClaimed });
      }
    }
    return result;
  }

  cleanup() {
    const now = Date.now();
    for (const [key, entry] of this._keys) {
      if (now - entry.lastActivity > this._idleTimeoutMs || now - entry.issuedAt > this._absoluteTimeoutMs) {
        this._keys.delete(key);
      }
    }
  }
}
```
