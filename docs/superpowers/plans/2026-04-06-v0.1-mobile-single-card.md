# Mobile Single-Card Terminal View Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a mobile-optimized terminal view that shows one session at a time, with touch gestures for scrolling/selection/switching, a text input bar with special keys, and voice input.

**Architecture:** A new `/mobile` route serves `mobile.html` which loads `mobile.mjs` and `mobile.css`. The mobile client connects to the existing `/ws/dashboard` WebSocket but only subscribes to one session at a time (unlike desktop which subscribes to all). A new server-side `unsubscribe` message type enables per-session unsubscription. The root route auto-redirects mobile User-Agents to `/mobile`. Terminal rendering reuses the existing `terminal.svg` via an `<object>` element.

**Tech Stack:** Node.js (server.mjs), vanilla JS (mobile.mjs), CSS (mobile.css), HTML, SVG `<object>` (terminal.svg), Web Speech API

**Files modified:**
- `/srv/svg-terminal/server.mjs` (~2030 lines) — add `unsubscribe` WS handler, `/mobile` route, mobile file routes, UA redirect, CLIENT_VERSION hash update
- `/srv/svg-terminal/mobile.html` (new) — mobile layout HTML
- `/srv/svg-terminal/mobile.mjs` (new) — mobile JS: WS connection, single-session subscription, gestures, input, voice
- `/srv/svg-terminal/mobile.css` (new) — mobile styles: full-viewport layout, touch areas, input bar

**a) `connectWs()`** — Same URL pattern as `connectDashboardWs()` (dashboard.mjs line 1510). Uses `location.protocol`, `location.host`, `/ws/dashboard`. No API key param (cookie auth only for mobile).

```javascript
function connectWs() {
  const proto = location.protocol === 'https:' ? 'wss:' : 'ws:';
  const url = proto + '//' + location.host + '/ws/dashboard';
  ws = new WebSocket(url);

  ws.onopen = function() {
    console.log('[Mobile WS] connected');
    reconnectAttempt = 0;
    // Do NOT subscribe to all sessions — wait for session-add events
    // and subscribe to one.
  };

  ws.onmessage = function(ev) {
    try {
      routeMessage(JSON.parse(ev.data));
    } catch (e) {
      console.warn('[Mobile WS] bad message', e);
    }
  };

  ws.onclose = function() {
    ws = null;
    currentSession = null;
    scheduleReconnect();
  };

  ws.onerror = function() {
    ws = null;
    currentSession = null;
  };
}
```

**b) `routeMessage(msg)`** — Handles:
- `version` — store, reload on mismatch (same as desktop line 1632)
- `reauth-required` — redirect to `/login`
- `session-add` — add to `sessions[]` array, update dropdown, do NOT auto-subscribe. After all initial session-adds arrive, subscribe to first (or last-used from localStorage).
- `session-remove` — remove from `sessions[]`, update dropdown, if current session removed switch to next.
- `screen` / `delta` — route to terminal.svg `<object>` via `terminalObj.contentWindow.renderMessage(msg)` (same pattern as desktop line 1726-1728).

```javascript
function routeMessage(msg) {
  if (msg.type === 'version') {
    if (serverVersion === null) {
      serverVersion = msg.version;
    } else if (serverVersion !== msg.version) {
      location.reload();
      return;
    }
    return;
  }

  if (msg.type === 'reauth-required') {
    location.href = '/login';
    return;
  }

  if (msg.type === 'session-add') {
    addSession(msg);
    return;
  }

  if (msg.type === 'session-remove') {
    removeSession(msg.session);
    return;
  }

  // Per-session messages (screen, delta, session-settings)
  if (msg.session && msg.session === currentSession) {
    if (msg.type === 'screen' || msg.type === 'delta') {
      routeToTerminalSvg(msg);
    }
  }
}
```

**c) `addSession(msg)`** — Server sends session-add for each discovered session on connect (sendSessionDiscovery, server.mjs line 748). Desktop auto-subscribes during discovery (line 757). Mobile does NOT — it just stores the session metadata.

```javascript
function addSession(msg) {
  if (sessions.find(s => s.name === msg.session)) return; // duplicate
  sessions.push({
    name: msg.session,
    cols: msg.cols || 80,
    rows: msg.rows || 24,
    title: msg.title || msg.session,
    source: msg.source || 'claude-proxy',
  });
  updateDropdown();

  // Auto-subscribe to first session if none selected
  if (!currentSession) {
    const lastUsed = localStorage.getItem('mobile-last-session');
    const target = (lastUsed && sessions.find(s => s.name === lastUsed))
      ? lastUsed : sessions[0].name;
    switchSession(target);
  }
}
```

**d) `switchSession(sessionName)`** — Unsubscribe old, subscribe new, animate.

```javascript
function switchSession(sessionName) {
  if (sessionName === currentSession) return;
  if (currentSession) {
    sendMessage({ type: 'unsubscribe', session: currentSession });
  }
  currentSession = sessionName;
  localStorage.setItem('mobile-last-session', sessionName);
  sessionDropdown.value = sessionName;

  // Subscribe — server will send back screen data
  sendMessage({
    type: 'subscribe',
    session: sessionName,
    source: 'claude-proxy',
  });
}
```

### Task 8: Implement gesture detection system

- [ ] In `mobile.mjs`, add a gesture handler on `#terminal-area`:

```javascript
const terminalArea = document.getElementById('terminal-area');
let touchState = null; // { startX, startY, startTime, mode: null|'scroll'|'select'|'swipe' }

terminalArea.addEventListener('touchstart', onTouchStart, { passive: false });
terminalArea.addEventListener('touchmove', onTouchMove, { passive: false });
terminalArea.addEventListener('touchend', onTouchEnd, { passive: false });

function onTouchStart(e) {
  if (e.touches.length !== 1) return;
  const t = e.touches[0];
  touchState = {
    startX: t.clientX,
    startY: t.clientY,
    startTime: Date.now(),
    mode: null,
    lastX: t.clientX,
    lastY: t.clientY,
  };
}

function onTouchMove(e) {
  if (!touchState || e.touches.length !== 1) return;
  e.preventDefault(); // prevent browser scroll/zoom
  const t = e.touches[0];
  const dx = t.clientX - touchState.startX;
  const dy = t.clientY - touchState.startY;
  const adx = Math.abs(dx);
  const ady = Math.abs(dy);

  if (!touchState.mode) {
    // Determine gesture mode
    if (adx > 20 && ady < 10) {
      touchState.mode = 'swipe';
    } else if (adx > 10 && touchState.mode !== 'scroll') {
      // Horizontal movement started first — could be selection
      touchState.mode = 'select';
    } else if (ady > 10 && adx < 10) {
      touchState.mode = 'scroll';
    }
  }

  if (touchState.mode === 'scroll') {
    handleScroll(t.clientY - touchState.lastY);
  } else if (touchState.mode === 'select') {
    handleSelection(t.clientX, t.clientY);
  }
  // swipe: no action during move, evaluated on touchend

  touchState.lastX = t.clientX;
  touchState.lastY = t.clientY;
}

function onTouchEnd(e) {
  if (!touchState) return;
  const dx = (touchState.lastX || 0) - touchState.startX;
  const elapsed = Date.now() - touchState.startTime;

  if (touchState.mode === 'swipe' && Math.abs(dx) > 100) {
    if (dx < 0) {
      switchToNextSession();
    } else {
      switchToPrevSession();
    }
  } else if (touchState.mode === 'select') {
    // Copy selection to clipboard
    copySelectionToClipboard();
  } else if (!touchState.mode && elapsed < 300) {
    // Tap — focus input field
    document.getElementById('text-input').focus();
  }

  touchState = null;
}
```

## Phase 7: Voice input

### Task 14: Implement voice input with Web Speech API

- [ ] In `mobile.mjs`:

```javascript
const voiceBtn = document.getElementById('voice-btn');
let recognition = null;
let isListening = false;

function initVoice() {
  const SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition;
  if (!SpeechRecognition) {
    voiceBtn.classList.add('unsupported');
    voiceBtn.title = 'Speech recognition not supported in this browser';
    return;
  }

  recognition = new SpeechRecognition();
  recognition.continuous = false;
  recognition.interimResults = true;
  recognition.lang = 'en-US';

  recognition.onresult = function(event) {
    let interim = '';
    let final = '';
    for (let i = event.resultIndex; i < event.results.length; i++) {
      const transcript = event.results[i][0].transcript;
      if (event.results[i].isFinal) {
        final += transcript;
      } else {
        interim += transcript;
      }
    }

    if (final) {
      textInput.value = final;
      textInput.style.color = ''; // normal color
    } else if (interim) {
      textInput.value = interim;
      textInput.style.color = '#888'; // gray for interim
    }
  };

  recognition.onend = function() {
    isListening = false;
    voiceBtn.classList.remove('listening');
    textInput.style.color = ''; // restore normal color
  };

  recognition.onerror = function(event) {
    console.warn('[Voice] error:', event.error);
    isListening = false;
    voiceBtn.classList.remove('listening');
  };
}

voiceBtn.addEventListener('click', function() {
  if (!recognition) return;

  if (isListening) {
    recognition.abort();
    isListening = false;
    voiceBtn.classList.remove('listening');
  } else {
    textInput.value = '';
    recognition.start();
    isListening = true;
    voiceBtn.classList.add('listening');
  }
});

// Initialize on load
initVoice();
```

## Known Limitations and Future Work

1. **Text selection in SVG `<object>`:** Gesture framework is in place but actual text selection within SVG is limited. May need to switch to inline SVG (option B) or HTML `<pre>` rendering (option D) for proper selection support.
2. **Keyboard resize handling:** When the mobile keyboard opens, the viewport shrinks. The current CSS uses `100dvh` which should adjust, but the terminal may need an explicit resize when keyboard appears/disappears. Track via `visualViewport.resize` event.
3. **PWA/installable:** Not included in v0.1. Can be added later with a service worker and manifest.json.
4. **Push notifications:** Not included. Badge-in-dropdown for activity on non-viewed sessions is a natural next step.
5. **Landscape orientation:** The fit-to-width logic works in portrait. Landscape auto-detection (fit-to-height) is deferred.
6. **Voice commands (stretch goal from spec):** "switch to [session]", "control c", etc. Not in v0.1.
